# NLP 워크플로우와 유사도 기반 챗봇

## 핵심

자연어를 다루려면 5단계 파이프라인이 필요함: **분절화 → 사전화 → 벡터화 → 패딩 → 임베딩**. 이 과정을 거쳐 자연어가 신경망이 처리 가능한 숫자로 변환된다.

## 토큰화 5단계

* **분절화** — 형태소(KoNLPy) 또는 공백 기반(Keras Tokenizer)으로 토큰 분리
* **사전화** — `fit_on_texts`로 토큰에 인덱스 부여 (빈도순)
* **벡터화** — `texts_to_sequences`로 문장을 인덱스 시퀀스로 변환
* **패딩** — `pad_sequences`로 길이 통일 (정보 보존 vs 메모리 트레이드오프)
* **임베딩** — 인덱스를 밀집 실수 벡터로 변환 (신경망 입력)

## 유사도 기반 챗봇 동작

1. 사용자 질문 입력 → 토큰화
2. 챗봇 시트의 모든 질문 벡터와 코사인 유사도 계산
3. 가장 높은 점수의 질문에 페어로 묶인 답변 반환

## Keras Tokenizer의 한계

인덱스 기반 벡터는 의미를 못 담음. 실제로 "오늘 비가 안 오네?"와 "내 지인한테 내 험담했대"가 유사도 0.9997로 매칭되는 문제 발생. 토큰 패턴이 우연히 일치한 케이스.

## SBERT로 교체

`sentence-transformers/xlm-r-100langs-bert-base-nli-stsb-mean-tokens` 모델로 토크나이저 교체. `model.encode()` 한 줄이 분절·사전·벡터화·임베딩 전 과정을 대체하며, 768차원 의미 임베딩을 출력. 토크나이저 한 줄 교체로 응답 품질이 크게 개선됨.

## 한 줄 회고

> 토큰화·벡터화·임베딩은 LLM API에 추상화되어 있지만, 데이터 파이프라인 관점에선 명확히 분리된 단계. 인덱스 시퀀스로 배치하면 의미가 사라지고, 사전 학습된 임베딩 공간에 배치하면 의미가 보존된다는 점이 핵심.
