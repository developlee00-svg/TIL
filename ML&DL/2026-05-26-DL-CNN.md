# CNN과 가중치 학습 파라미터에 대한 통찰

CNN의 합성곱 연산을 PyTorch로 직접 구현하고 `nn.Conv2d` API와 결과를 비교하여 "학습 = 커널 가중치 자동 최적화"임을 확인한 학습 노트입니다.

---

## 핵심 요약

### 1. CNN 개요

사람·동물의 눈이 사물을 인지하는 절차를 차용. 이미지·영상 데이터의 인지·분류·탐지에 적합한 모델로, 비전 계열 신경망의 출발점.

### 2. CNN 전체 구조

```
입력층
------------
은닉층
  ├─ 합성곱층 : 이미지 특징 추출
  ├─ 풀링층  : 특징 강화
  ├─ ...반복
  └─ 전결합층 : 차원 조정 (2D → 1D)
------------
출력층
```

### 3. 입력 데이터 포맷

| 포맷 | 순서 | 프레임워크 |
| --- | --- | --- |
| NHWC | 개수-높이-너비-채널 | TensorFlow (Channel Last) |
| NCHW | 개수-채널-높이-너비 | PyTorch (Channel First) |

> 프레임워크 전환 시 차원 재배열 필수. CNN 디버깅 최다 실수 지점.

### 4. 합성곱층 구성 요소

- **커널(필터) k** — 가중치를 값으로 가지는 행렬 (정방형)
- **stride(s)** — 커널 이동량
- **padding(p)** — 경계 보정 (SAME: 원본 크기 유지, VALID: 작아짐)
- **출력** — Feature Map → 활성화함수 통과 → Activation Map

### 5. 풀링층 vs 합성곱층

| 항목 | 합성곱층 | 풀링층 |
| --- | --- | --- |
| 학습 파라미터 | ✅ 있음 (커널 가중치) | ❌ 없음 |
| 역할 | 특징 추출 | 특징 강화 |
| 기법 | 행렬곱 | Max / Average |

### 6. PyTorch 직접 구현 (수직 커널)

```python
ver_kernel = torch.Tensor([
    [1, 0, -1],
    [1, 0, -1],
    [1, 0, -1]
])

def cus_convolution(x, kernel, stride):
    img_h, img_w = x.size()
    k_h, k_w = kernel.size()
    final_buf = []
    for i in range(0, img_h - k_h + 1, stride):
        feature_map_buf = []
        for j in range(0, img_w - k_w + 1, stride):
            img_target = x[i:i+k_h, j:j+k_w]
            feature_map_buf.append(torch.sum(img_target * kernel))
        final_buf.append(feature_map_buf)
    return torch.Tensor(final_buf)
```

- 원본 (150, 150) → Feature Map (148, 148) (VALID 모드)
- `-1` 때문에 음수 발생 → 시각화 시 노이즈 → **ReLU로 보정** (`torch.relu(feature_map)`)

### 7. `nn.Conv2d` API와 비교

```python
import torch.nn as nn

conv_layer = nn.Conv2d(1, 1, 3, 1, bias=False)
# 가중치 수동 주입 (학습 대신)
conv_layer.weight.data = ver_kernel.expand(1, 1, 3, 3)

cur_img = img_tensor.expand(1, 1, 150, 150)   # NCHW로 변환
feature_map = conv_layer(cur_img)

torch.sum(feature_map == ver_feature_map)     # 148*148 완전 일치
```

- `weight.data`에 사람이 직접 가중치 주입 = 학습 결과를 흉내내는 행위
- **학습이란 이 weight.data를 자동으로 튜닝하는 과정**

### 8. 핵심 인사이트

> 사진 앱의 "필터"가 곧 합성곱 커널이다. 차이는 사진 앱은 **사람이 설계한 고정 커널**, CNN은 **학습으로 최적 커널을 찾는 것**뿐이다. for-loop 직접 구현 + `nn.Conv2d.weight.data` 수동 주입을 해보면 **"학습된 모델 = 최적의 커널들을 찾아낸 모델"** 이라는 본질이 명확해진다.
